## ⚡ What is DispatchMindAI?

**DispatchMindAI** is an autonomous decision engine for **RedX**, Bangladesh's leading last-mile logistics platform. It uses a multi-agent AI pipeline powered by **Claude Sonnet 4.6** to make real-time dispatch decisions — choosing between internal fleet (3PL) and external delivery partners (4PL) based on live cost modeling, SLA risk analysis, and hub profitability projections.

> *"Stop guessing. Start dispatching intelligently."*

---

## 🧠 The AI Agent Pipeline

```
 Parcel In ──►  AGENT 1        AGENT 2         AGENT 3
               ┌──────────┐   ┌─────────────┐  ┌──────────────┐
               │ Volume   │──►│ Cost        │  │ SLA Risk     │
               │ Forecast │   │ Modeling    │  │ Analyzer     │
               └──────────┘   └──────┬──────┘  └──────┬───────┘
                                     │                 │
                                     ▼                 ▼
                               AGENT 4: Partner Evaluation
                              ┌─────────────────────────────┐
                              │  Ranks partners by margin   │
                              │  uplift × SLA reliability   │
                              └───────────────┬─────────────┘
                                              │
                              ┌───────────────▼─────────────┐
                              │   AGENT 6: Executive        │
                              │   Summary (plain English)   │
                              └───────────────┬─────────────┘
                                              │
                                    ◄─────────┘
                              Dispatch Decision + Report
```

Each agent queries a **pre-aggregated MySQL database**, calls **Claude Sonnet 4.6**, and passes its structured output to the next agent in the chain.

## 🎯 Features

| Feature | Description |
|---------|-------------|
| 🚚 **Smart Dispatch** | AI decides 3PL vs 4PL per shipment based on margin and SLA risk |
| 🤝 **Partner Optimizer** | Ranks delivery partners by SLA breach probability × cost efficiency |
| 📊 **Hub Profitability** | Predicts 90-day margin and recommends keep / close / convert |
| 🏗️ **Model Advisor** | Compares 3PL-only vs 4PL-only vs Hybrid operating models |
| 🏥 **Hub Health** | Fleet-wide profitability scan — identifies losing hubs + suggests partner reassignments |
| ↔️ **Partner Assignment** | One-click area-to-partner reassignment writes directly to the database |
| 📝 **Executive Summary** | Plain-English report generated by Claude for operations managers |
| 🕓 **Decision History** | In-memory audit trail of last 200 dispatch decisions |

## 🚀 Quick Start

### Prerequisites
- Node.js 22+
- Docker
- Anthropic API key
- Access to `stage1_shopuplite` DB (via Teleport)

### Setup

```bash
# 1. Install dependencies
npm install

# 2. Configure environment
cp .env.example .env
# Fill in DB_*, STAGE_DB_*, ANTHROPIC_API_KEY, JWT_SECRET

# 3. Start local database
npm run db:up

# 4. Sync data from stage
npm run db:sync

# 5. Seed dummy hub costs (first time)
curl -X POST http://localhost:3000/api/v1/hubs/129/costs \
  -H "Content-Type: application/json" \
  -d '{"year":2026,"month":2,"rent":80000,"employee_cost":150000,"utility_cost":15000,"maintenance_cost":10000,"other_cost":5000}'

# 6. Populate aggregation tables
npm run db:aggregate

# 7. Start the server
npm run dev
```

## 📡 API Reference

### `POST /api/v1/dispatch/recommend`
Runs the full 5-agent pipeline. Returns a dispatch decision + executive summary.
`hub_id` is optional — derived automatically from `area_id` if not provided.

```json
// Request
{
  "area_id": 91,
  "parcel_value": 1500,
  "weight": 1.2,
  "sla_days": 3
}

// Response
{
  "type": "4PL",
  "partner": "Pathao",
  "expected_margin": 57.43,
  "risk_score": 28,
  "confidence": 74,
  "summary": "Dispatch Decision Report | Hub 2 | Area 91 ..."
}
```

> ⏱️ Expect 15–30 seconds — this calls Claude 5 times.

### `GET /api/v1/partners/optimize?area_id=91`
Runs SLA Risk + Partner Evaluation agents. Hub derived server-side.

### `GET /api/v1/hubs/summary`
Fleet-wide hub health analysis. Runs a single Claude call across the 15 worst-performing hubs.
Returns per-hub recommendations + specific area-level suggested partner assignments.

### `POST /api/v1/hubs/assign-partners`
Writes area-to-partner assignments to `sl_area_partners`.

```json
{ "assignments": [{ "area_id": 91, "partner_id": 11 }] }
```

### `GET /api/v1/hubs/:hubId/profitability`
Runs Volume Forecast + Cost Modeling + Network Strategy agents.

### `GET /api/v1/hubs/:hubId/model-advice`
Compares 3PL-only vs 4PL-only vs Hybrid models for the hub.

### `GET /api/v1/dispatch/history`
Returns the last 200 dispatch decisions from in-memory store.

### `GET /api/v1/hubs` · `GET /api/v1/areas`
Hub and area lists for UI dropdowns.

## 🗄️ Database Design

### Aggregation Tables (local, populated by `npm run db:aggregate`)

| Table | Purpose |
|-------|---------|
| `dm_hub_daily_volume` | Pre-aggregated parcel counts per hub per day (90d) |
| `dm_partner_sla_performance` | Breach rate per partner/area/month (6 months) |
| `dm_hub_contribution_margin` | Revenue, 4PL cost, fixed cost, margin per hub/month |
| `dm_hub_monthly_costs` | Manual cost inputs (rent, salaries, utilities) |

### Key Tables Synced from Production

| Table | Purpose |
|-------|---------|
| `sl_area_partners` | Maps delivery areas to 4PL courier partners |
| `sl_fourpl_partner_pricing` | Zone-aware weight-tier pricing per 4PL partner |
| `sl_shop_zones` | Shop-level delivery pricing by zone |

**Zone mapping:** `sl_areas.ZONE_ID` 1=ISD (Dhaka City), 2=SUB (Suburbs), 7+=OSD (Outside Dhaka)
**3PL vs 4PL:** `sl_delivery_partners.ID = 3` is Shopup Internal (3PL). All other partner IDs are 4PL.

## 🧪 Running Demo Scenarios

Open `requests.http` in VS Code with the [REST Client](https://marketplace.visualstudio.com/items?itemName=humao.rest-client) extension, or import `postman_collection.json` into Postman.

**Recommended demo flow:**
1. `GET /health` — verify server is up
2. `GET /hubs/summary` — Hub Health: fleet overview, losing hubs, suggestions
3. `POST /dispatch/recommend` — flagship AI decision (area 91, 1.2kg, BDT 1500)
4. `GET /partners/optimize?area_id=2` — partner ranking for an area
5. `GET /hubs/2/profitability` — hub keep/close/convert
6. `GET /dispatch/history` — audit trail
